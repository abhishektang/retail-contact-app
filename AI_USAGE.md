# AI Usage Disclosure

## Which AI tools you used

I built this application using a custom **agile AI team** — an orchestrated, multi-agent setup where each "team member" is a distinct role backed by a different LLM, each with its own rules and responsibilities defined. The roles mirror a real software team:

| Role | Responsibility |
|------|----------------|
| Product Owner | Triages requirements, defines acceptance criteria, reviews completed work |
| Technical Lead | Produces the technical plan, reviews code, approves/rejects implementations |
| Scrum Master | Converts requirements into user stories, assigns work, estimates effort |
| Senior Developer | Implements complex features and architectural changes |
| Junior Developer | Implements small fixes and isolated tasks |
| Senior Tester | Designs integration tests and test strategy |
| Junior Tester | Writes unit tests for isolated functions |
| DevOps Engineer | Handles branching, pushing code, and pull requests |

Each role is powered by a different model, chosen to match the complexity and cost of its work. The underlying models spanned **Claude Sonnet 4.6, Claude Opus 4.6, GPT-5.4, and Minimax 2.7**.

## Why you chose those tools and what you used each of them for

I chose this orchestration approach because routing each task to the most appropriate model **increases efficiency and reduces token usage and cost**. The division of labour follows a clear principle: **high-level (high-capability) models are used for planning, reviewing, and suggesting, while low-level (cheaper) models implement the high-level models' feedback.** Expensive, high-capability models like Opus and GPT drive the high-value thinking — architecture, technical planning, and code review — and the cheaper models carry out the resulting implementation work (writing code, small fixes, unit tests, boilerplate). This keeps the overall cost down without sacrificing quality on the parts that matter.

I used the team to:

- Scaffold the **Next.js** client and **NestJS** server projects
- Write the initial implementation code across frontend and backend
- Install and configure dependencies
- Write the test cases (unit and end-to-end)

## Which parts were generated by the tool, and which parts I wrote, modified, or designed myself

Most of the **initial code** was generated by the AI team. My own role was substantial and hands-on throughout:

- **Architecture** — I defined the overall system architecture and data flow.
- **Database design** — I designed the database schema and chose PostgreSQL over SQLite for production readiness.
- **Debugging and fixing** — I debugged the code and fixed it wherever it failed.
- **In-the-loop guidance** — I stayed part of the AI generation loop, giving corrective code suggestions whenever the AI produced wrong output or hallucinated, and steered it back on track.

In short, the AI generated the bulk of the boilerplate and first-pass code, while I owned the design decisions, validated the logic, and corrected the AI's mistakes.

## Anything else you used alongside the AI tools

- **Official framework documentation** — to understand the specific methods and APIs of each framework.
- **VS Code** — code editor.
- **Docker Desktop** — for containerisation and local testing.
